## 4. Architecture decisions

### App shell flow
`App.tsx` keeps `<Canvas>` always mounted (game world stays loaded). Screens are `position: absolute` overlays at `z-index: 100+`. This avoids scene teardown/reload cost when entering/exiting menus.

### No router
No routing library installed. The `appShellStore` provides all screen transitions with guarded actions (`pauseGame` only transitions from `'playing'`, `resumeGame` only from `'paused'`).

### Styling approach
Consistent with existing codebase: inline `React.CSSProperties` for all component styles. A single `<style>` tag per screen component injects `@keyframes` and media-query overrides for animations. No Tailwind, no CSS modules.

### Accessibility implementation
- **CSS-first**: `@media (prefers-reduced-motion: reduce)` suppresses all animations globally at the stylesheet level.
- **JS override**: When the user sets `reducedMotion: true`, `PrefsBodySync` adds `body.reduced-motion` which triggers the same suppression block via a class selector.
- **Persistence**: prefs stored in `localStorage` under `'echoes-prefs'` via Zustand `persist` middleware ā€” survive page reloads.
- **Focus management**: `PauseMenuScreen` and `CreditsScreen` use `autoFocus` on their primary action buttons. `PauseMenuScreen` is marked `role="dialog" aria-modal="true"`.

### Palette
All new screens use the existing dark palette: `#09030f` base, `#e8c8e0` (purple/pink) for primary text and title, `#bfe8ff` (blue) for secondary/tertiary, `rgba(10,12,18,ā€¦)` for panel backgrounds.

## 6. Known limitations / follow-ups

- **Focus trap**: pause and credits screens do not trap keyboard focus ā€” a future accessibility pass could add `focus-trap-react` or a custom trap hook.
- **Camera transition + reduced motion**: `cameraStore.ts` transition duration is a fixed config value. A future pass could subscribe to `useEffectiveReducedMotion()` in `GameCamera.tsx` and snap transitions instantly when reduced motion is active.
- **`body.reduced-motion` media query selector**: `@media (body.reduced-motion)` is not valid CSS ā€” the suppression block uses a separate class selector rule instead (`body.reduced-motion *, ā€¦`). Both the OS media query path and the JS class path work correctly.
- **Main menu onboarding tips**: a first-run tooltip or WASD hint on the main menu (or immediately after `startGame()`) was deferred ā€” can be added as a `features/ui/OnboardingOverlay.tsx` without touching any other file.
